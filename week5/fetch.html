How to use Post method

      // Define the user object
      let user = {
        name: 'John',
        surname: 'Smith'
      };
  
      // Make a POST request to the server
      let response = await fetch('/article/fetch/post/user', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json;charset=utf-8'
        },
        body: JSON.stringify(user)
      });
  
      // Retrieve the response data as JSON
      let result = await response.json();
  
      // Display the result message
      alert(result.message);

In order to make a POST request or any other HTTP request using the fetch API, we can specify the method and include the request body using the method and body options respectively. For instance, when sending JSON data, we can create an object representing the data, stringify it using JSON.stringify(), and set the Content-Type header to application/json;charset=utf-8. After making the request, we can handle the response by using response.json() to parse the response body as JSON. Finally, we can access the desired data from the response and perform further actions as needed.

Fetch: Cross-Origin Requests

CORS (Cross-Origin Resource) is a mechanism that allows web browsers to enforce security policies regarding cross-origin requests made by web pages or web applications.
An "origin" refers to the combination of three components: the protocol, the domain, and the port, which together identify the source of a web page or a web application. The origin represents the specific location from which a web resource is served.
  • Protocol: The protocol helps determine the security context of the resource. The common protocols include HTTP (Hypertext Transfer Protocol) and HTTPS (HTTP Secure).
  • Domain: The domain is the name or IP address that uniquely identifies a website or web application. It indicates the location where the web resource is hosted. For example: "https://www.example.com," the domain is "example.com."
  • Port: The port is the endpoint within a server where the web resource is being served. (e.g., localhost:8080).

Rest parameters and spread syntax

The "..." in JavaScript has two uses: rest parameters and spread syntax.
  • Rest parameters gather multiple function arguments into an array and enable functions to accept any number of arguments
  • Spread Syntax expands an array into individual elements and simplifies working with arrays by spreading their elements

How Single-Page Applications Work

A Single-Page Application (SPA) is a web application that loads on a single page and updates the content without refreshing the entire page. This provides a smoother user experience and avoids the page load times between pages.
